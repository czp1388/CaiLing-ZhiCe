# 彩灵·智策 — 推荐算法改进方案

## 一、现状诊断

### 当前推荐流程
```
热号Top15 → 综合评分(排名+遗漏+开出率) → 五膽拖胆码(随机采样300次) → 合并去重 → 6个推荐号码
```

### 当前推荐结果示例（seed=42）
```
推荐号码: [2, 13, 20, 28, 37, 46]
平均开出率: 15.2% (预期14.3%, 偏差+0.9%)
信心评级: 中
策略: 热号6个 + 五膽拖胆码5个
```

### 现存问题

#### 问题1：维度单一，仅依赖热号+五膽拖
- 推荐结果 = 5个胆码（全部热号）+ 1个额外热号
- **6个推荐号码全是热号**，没有冷号，没有遗漏反弹考虑
- 当前推荐 `2,13,20,28,37,46` 全部是热号，0个冷号
- 虽然热的逻辑合理但**全热不现实**，历史上全热号组合的成功率远低于冷热搭配
- 没有区间分布约束（可能6个全在同一个区间）
- 没有奇偶比、大小比、和值范围约束

#### 问题2：信心评级标准不合理
- "高"需要平均偏差 ≥ +3%，即平均开出率 ≥ 17.3%
- 但**实际数据中，单号码最大开出率仅为16.4%**（号码11和12）
- 所有49个号码的开出率分布在11.3%~16.4%之间
- 均值14.3%，标准差仅1.4%
- 最大z-score = +1.56（号码11, 12）
- **结论：在当前标准下，"高"置信度理论上永远不可能达到**

| 号码 | 开出率 | z-score | 当前评级应为 |
|------|--------|---------|-------------|
| 11 | 16.4% | +1.56 | 较高 |
| 12 | 16.4% | +1.56 | 较高 |
| 2 | 16.2% | +1.37 | 较高 |
| 32 | 16.2% | +1.37 | 较高 |
| 7 | 15.9% | +1.17 | 中~较高 |
| 10 | 11.3% | -2.21 | 低 |

#### 问题3：五膽拖回测验证不充分
- `auto_backtest` 仅从热号Top15中随机采样300次
- 没有真正的滚动回测（模拟历史逐期推荐，看实际命中率）
- 胆码全是热号，组合单一
- 没有对比随机基线（随机选6个的期望命中数 ≈ 0.86个/期）

#### 问题4：综合评分公式过于简单
```python
score = (15 - hot_idx) * 3 + miss_val * 0.5 + hit_rate * 0.3
```
- 热号排名权重（×3）远大于遗漏值（×0.5）和开出率（×0.3）
- 没有号码组合共现分析
- 没有K线技术指标的融合（虽然已经计算了KDJ/RSI/MACD等，但没用在推荐中）
- 没有区间/奇偶/大小约束

#### 问题5：遗漏值利用方式原始
- 直接使用原始遗漏值（0-100范围），未做归一化
- 大遗漏值的权重被低估（×0.5）
- 遗漏值应当和热号分析**结合**：热号+大遗漏 的优先级最高

#### 问题6：只有6个正选推荐，没有胆拖组合
- 对于六合彩，6个号码全中头奖的概率极低（1/13,983,816）
- 更实用的策略是推荐胆拖组合（选3-4个胆码+拖码），提高中奖概率
- 胆拖模式下，胆码中3个以上即有奖

### 改进1：新增多维度分析引擎（优先级：P0）

#### 1.1 区间分布分析（1-12, 13-24, 25-36, 37-49）

六合彩6个正选号的经典分布规律：4个区间各1-2个，极端分布（3+个在同一区间）概率低。

```python
ZONE_RANGES = [(1,12), (13,24), (25,36), (37,49)]

def zone_distribution(numbers):
    """计算一组号码的区间分布
    返回: [区间1个数, 区间2个数, 区间3个数, 区间4个数]
    """
    zones = [0, 0, 0, 0]
    for n in numbers:
        for i, (lo, hi) in enumerate(ZONE_RANGES):
            if lo <= n <= hi:
                zones[i] += 1
                break
    return zones
```

#### 1.2 奇偶比分析

```python
def odd_even_ratio(numbers):
    """返回 (奇数个数, 偶数个数)"""
    odd = sum(1 for n in numbers if n % 2 == 1)
    return (odd, len(numbers) - odd)
```

历史分布：3:3 最常见（约35%），4:2 次之（约30%），2:4 约25%，5:1 和 1:5 罕见。

#### 1.3 大小比分析

```python
def big_small_ratio(numbers, threshold=25):
    """大小比：大于等于threshold为大，小于为小"""
    big = sum(1 for n in numbers if n >= threshold)
    return (big, len(numbers) - big)
```

#### 1.4 和值范围分析

```python
def sum_of_six(numbers):
    """6个正选号的和值"""
    return sum(numbers[:6])  # 只算6个正选，不含特别号
```

历史分布：平均值约150，范围通常在100-200之间。

#### 1.5 号码组合共现分析

```python
def pair_cooccurrence(window=200):
    """分析号码对共现频率
    返回: [( (a,b), 共现次数 ), ...]
    """
    conn = get_db()
    rows = conn.execute(
        "SELECT n1,n2,n3,n4,n5,n6 FROM draws ORDER BY draw_date DESC LIMIT ?",
        (window,)
    ).fetchall()
    conn.close()

    pairs = defaultdict(int)
    for r in rows:
        nums = [r["n1"], r["n2"], r["n3"], r["n4"], r["n5"], r["n6"]]
        for i in range(len(nums)):
            for j in range(i+1, len(nums)):
                pair = tuple(sorted([nums[i], nums[j]]))
                pairs[pair] += 1

    return sorted(pairs.items(), key=lambda x: -x[1])
```

### 改进2：重构综合评分模型（优先级：P0）

**位置：** 新增 `core/scorer.py` 或直接重写 `core/recommender.py`

新的评分模型应融合以下维度（归一化到0-100）：

```python
FEATURE_WEIGHTS = {
    "hot_score":      0.20,  # 热号排名百分位
    "omission_score": 0.25,  # 遗漏值百分位（越高=越久未出→反弹概率越高）
    "kline_score":    0.15,  # K线技术指标信号（KDJ/RSI超卖反弹）
    "pair_score":     0.15,  # 与已选号码的共现度
    "zone_score":     0.10,  # 区间平衡贡献
    "odd_even_score": 0.10,  # 奇偶平衡贡献
    "big_small_score":0.05,  # 大小平衡贡献
}

def calculate_number_score(num, context, features_db):
    """
    num: 待评分的号码
    context: 已选号码列表（用于计算共现/区间平衡）
    features_db: 预计算的特征数据库
    
    返回: 0-100的综合得分
    """
    # 1. 热号得分（基于频率百分位）
    hot_score = features_db.get_hot_percentile(num)
    
    # 2. 遗漏值得分（归一化百分位，越高越好）
    omission_score = features_db.get_omission_percentile(num)
    
    # 3. K线技术信号
    kline_score = features_db.get_kline_signal_score(num)
    
    # 4. 共现得分（与已选号码的平均共现频率）
    pair_score = features_db.avg_pair_cooccurrence(num, context)
    
    # 5. 区间平衡得分（加入此号码后区间分布更均衡则得分高）
    zone_score = features_db.zone_balance_score(num, context)
    
    # 6. 奇偶平衡得分
    oe_score = features_db.odd_even_balance_score(num, context)
    
    # 7. 大小平衡得分
    bs_score = features_db.big_small_balance_score(num, context)
    
    scores = {
        "hot_score": hot_score,
        "omission_score": omission_score,
        "kline_score": kline_score,
        "pair_score": pair_score,
        "zone_score": zone_score,
        "odd_even_score": oe_score,
        "big_small_score": bs_score,
    }
    
    total = sum(scores[k] * FEATURE_WEIGHTS[k] for k in FEATURE_WEIGHTS)
    return round(total, 1), scores
```

### 改进3：重做信心评级标准（优先级：P0）

#### 当前问题数据验证

```
所有49个号码的整体开出率统计（基于755期）：
  均值: 14.3% (理论值 = 7/49)
  标准差: 1.4%
  最大值: 16.4% (号码11, 12 → z=+1.56)
  最小值: 11.3% (号码10 → z=-2.21)
  
结论：最大偏差仅+2.1%，远不及当前"高"要求的+3%
```

#### 方案A：基于z-score（推荐）

```python
def calculate_confidence_zscore(avg_hit_rate, all_rates):
    """
    基于z-score的信心评级
    
    参数:
        avg_hit_rate: 推荐6个号码的平均开出率
        all_rates: 所有49个号码的历史开出率列表
    
    评级标准:
        z >= +1.5 → "高"（前7%最优）
        z >= +0.5 → "较高"（前30%）
        z >= -0.5 → "中"（中间38%）
        z >= -1.5 → "较低"（后30%）
        z <  -1.5 → "低"（后7%）
    """
    import statistics
    mean = statistics.mean(all_rates)
    std = statistics.stdev(all_rates)
    z = (avg_hit_rate - mean) / std
    
    if z >= 1.5:
        return "高", z
    elif z >= 0.5:
        return "较高", z
    elif z >= -0.5:
        return "中", z
    elif z >= -1.5:
        return "较低", z
    else:
        return "低", z
```

### 改进4：新增滚动回测验证模块（优先级：P1）

**位置：** 扩展 `core/backtest.py`

#### 4.1 滚动回测算法

```python
def rolling_backtest(recommender_fn, window=100, test_periods=50):
    """
    滚动回测：模拟过去N期的推荐决策过程
    
    对每一期 t（从window到window+test_periods-1）：
    1. 用前 window 期数据训练模型
    2. 输出推荐号码（6个正选 + 胆拖方案）
    3. 检查第 t 期的实际开奖结果
    
    返回回测统计结果
    
    参数:
        recommender_fn: 推荐函数，接收 (seed, train_window) 参数
        window: 训练窗口大小
        test_periods: 回测期数
    """
    conn = get_db()
    all_rows = conn.execute(
        "SELECT draw_date,n1,n2,n3,n4,n5,n6,extra FROM draws ORDER BY draw_date"
    ).fetchall()
    conn.close()
    
    results = []
    t_end = min(window + test_periods, len(all_rows) - 1)
    
    for t in range(window, t_end):
        # 第t期的实际开奖
        actual = all_rows[t]
        actual_nums = {actual["n1"], actual["n2"], actual["n3"],
                       actual["n4"], actual["n5"], actual["n6"]}
        
        # 模拟：用第t期之前的数据做推荐
        # 注意：这里需要一种方式让recommender只用到前t期数据
        # 可以传入 train_limit=t 参数
        rec = recommender_fn(seed=42, train_limit=t)
        recommended = set(rec["numbers"])
        
        # 命中统计
        hits = len(recommended & actual_nums)
        # 胆码命中
        cores = set(rec.get("drag_cores", []))
        core_hits = len(cores & actual_nums) if cores else 0
        
        results.append({
            "date": actual["draw_date"],
            "recommended": sorted(recommended),
            "actual": sorted(actual_nums),
            "hits": hits,
            "hit_rate_6": round(hits / 6 * 100, 1),
            "core_hits": core_hits,
            "any_hit": hits >= 1,
            "two_plus": hits >= 2,
        })
    
    return aggregate_backtest_results(results)
```

#### 4.2 核心评估指标

| 指标 | 公式 | 说明 | 随机基线 |
|------|------|------|---------|
| 平均命中数 | Σ(hits)/N | 每期6个推荐中几个 | 0.86 |
| 命中率 ≥1 | Σ(any_hit)/N | 至少中1个的期占比 | 60% |
| 命中率 ≥2 | Σ(two_plus)/N | 至少中2个的期占比 | 20% |
| 胆码命中 ≥2 | Σ(core_hits≥2)/N | 胆码方案的有效性 | — |

#### 4.3 随机基线计算

```
6个正选号 + 1个特别号 = 7个中奖号码
从49个中随机选6个，每个号码中奖概率 = 7/49 ≈ 14.3%
期望命中数 = 6 × 7/49 ≈ 0.857个/期
期望命中率 ≥1 = 1 - C(42,6)/C(49,6) ≈ 60%
期望命中率 ≥2 = ...
```

**回测目标：推荐算法平均命中数 > 0.86，且置信区间不包含0.86**

#### 5.2 胆拖组合生成算法

```python
def generate_drag_strategy(scored_numbers, n_cores=4, n_drags=5):
    """
    从评分最高的号码中生成胆拖组合
    
    胆码选择策略：
    1. 胆码：综合评分前n_cores
    2. 必须覆盖至少2个区间
    3. 奇偶比 2:2 或 3:1
    4. 大小比近似均衡
    
    拖码选择策略：
    1. 胆码之外的号码中评分最高的n_drags个
    2. 优先选择能和胆码覆盖更多区间的号码
    3. 包含至少1个遗漏值高的冷号
    """
    sorted_nums = sorted(scored_numbers.items(), key=lambda x: -x[1]["total_score"])
    
    # 选胆码（带约束）
    cores = []
    for num, features in sorted_nums:
        if len(cores) >= n_cores:
            break
        test_cores = cores + [num]
        # 约束检查
        zones = zone_distribution(test_cores)
        if max(zones) <= 2:  # 同一区间不超过2个
            cores.append(num)
    
    # 选拖码
    used = set(cores)
    candidates = [(n, f) for n, f in sorted_nums if n not in used]
    
    # 优先级：评分 + 区间覆盖 + 遗漏值
    drags = []
    needed_zones = get_uncovered_zones(cores)
    for num, features in candidates:
        if len(drags) >= n_drags:
            break
        drags.append(num)
        # 可以考虑加权排序而不是简单取前N个
    
    return {"cores": cores, "drags": drags}
```

### 改进6：遗漏值+热号 融合分析（优先级：P0）

核心原则：**热号 + 大遗漏 = 优先级最高的号码**

实现方式：

```python
def hot_omission_priority(hot_nums, omission_dict):
    """
    热号与遗漏值融合排序
    
    逻辑：
    1. 从热号Top15中筛选出遗漏值 ≥ 阈值的号码
    2. 对冷号（非热号）中遗漏值极大（前10%）的特殊处理
    3. 排序：热号+大遗漏 → 热号+中等遗漏 → 冷号+极大遗漏
    
    返回: 综合优先级排序的号码列表
    """
    hot_set = set(hot_nums)
    max_omission = max(omission_dict.values())
    
    priority_scores = {}
    for num in range(1, 50):
        omission = omission_dict.get(num, 0)
        omission_norm = omission / max_omission * 100  # 归一化0-100
        
        is_hot = num in hot_set
        
        if is_hot:
            # 热号：遗漏越高越好（反弹潜力 + 热度支撑）
            base = 80 if omission_norm >= 50 else 60 if omission_norm >= 20 else 50
            hot_rank = hot_nums.index(num) if num in hot_nums else 50
            hot_score = max(0, 100 - hot_rank * 5)
            score = base * 0.4 + hot_score * 0.3 + omission_norm * 0.3
        else:
            # 冷号：只有遗漏极高（前10%）才考虑
            if omission_norm >= 90:
                score = omission_norm * 0.8  # 纯粹遗漏反弹逻辑
            elif omission_norm >= 70:
                score = omission_norm * 0.5
            else:
                score = 0  # 普通冷号不考虑
        
        priority_scores[num] = round(score, 1)
    
    return sorted(priority_scores.items(), key=lambda x: -x[1])
```

### 改进7：技术指标信号融合（优先级：P2）

**位置：** 新增 `core/indicator_signal.py`

`kline.py` 已经计算了以下技术指标但未被推荐引擎使用：
- KDJ（随机指标）：超卖区（K<20）→ 反弹信号
- RSI（相对强弱）：超卖（RSI<30）→ 即将反弹
- MACD：金叉（DIF上穿DEA）→ 趋势转强
- BOLL（布林带）：触下轨→ 反弹支撑
- ADX（趋势强度）：ADX>25 → 趋势确认

```python
def get_technical_score(number, window=200):
    """
    获取号码的技术面综合得分（0-100）
    
    信号类型与权重：
    - 超卖反弹（KDJ/RSI）：权重0.6
    - 趋势确认（MACD/ADX）：权重0.3
    - 布林带支撑：权重0.1
    """
    data = build_kline_data(number, window)
    if "error" in data:
        return 50  # 中性默认值
    
    ind = data["indicators"]
    signals = []
    
    # KDJ信号
    k, d, j = ind["kdj"]
    if k and d and k[-1] is not None and d[-1] is not None:
        if k[-1] < 20 and d[-1] < 20:
            signals.append(("kdj_超卖", 80))
        elif k[-1] > d[-1] and k[-2] <= d[-2]:
            signals.append(("kdj_金叉", 70))
    
    # RSI信号
    rsi = ind["rsi"]
    if rsi and rsi[-1] is not None:
        if rsi[-1] < 30:
            signals.append(("rsi_超卖", 85))
        elif rsi[-1] < 40 and rsi[-1] > rsi[-2]:
            signals.append(("rsi_回升", 60))
    
    # MACD信号
    dif, dea, macd_hist = ind["macd"]
    if dif and dea and dif[-1] is not None and dea[-1] is not None:
        if dif[-1] > dea[-1] and dif[-2] <= dea[-2]:
            signals.append(("macd_金叉", 70))
    
    # 综合得分
    if not signals:
        return 50
    avg_signal = sum(s[1] for s in signals) / len(signals)
    return round(avg_signal, 1)
```

## 五、关键注意事项

1. **六合彩本质是随机游戏** — 任何推荐算法都无法保证中奖。改进目标是**实现统计意义上的微弱优势**（选号决策质量高于完全随机）。

2. **冷热号定义需动态更新** — 建议每20期重新计算冷热号划分。窗口大小（当前100期）可以调整为50期（更敏感）或200期（更稳定）。

3. **回测过拟合风险** — 滚动回测必须严格区分训练集和测试集。不要在全部历史数据上调参后返回"高命中率"——那一定是过拟合。

4. **API向后兼容** — 改进后 `get_recommendation()` 的返回格式应尽量向后兼容，新增字段而非删除字段。现有使用者（cli.py, gui/app.py）不应报错。

5. **数据量限制意识** — 755期历史数据统计学意义有限（每号码平均出现约108次）。统计检验的置信区间较宽，2%以下的差异不显著。

6. **与预期值（EV）模块的协同** — 推荐算法可以调用 `core/ev.py` 的 `calculate_ev()` 来评估推荐号码的理论期望值，作为辅助参考。

7. **记录推荐版本号** — 在输出中添加 `recommender_version: "2.0"` 字段，方便后续对比不同版本的效果。
